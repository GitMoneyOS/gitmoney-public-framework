# Contributing

## Purpose

Keep GitMoney changes reviewable, plain-English, and connected to business control.

## Branch Naming

Use short names like:

- `docs/audit-template`
- `setup/repo-baseline`
- `content/case-file-draft`
- `saas/score-model`

## Pull Request Checklist

- [ ] The business change is clear.
- [ ] The owner is named.
- [ ] The evidence is attached or described.
- [ ] No secrets, client private data, or unapproved claims are included.
- [ ] AI-generated work was reviewed by a human owner.

## Review Expectation

Use pull requests for important changes. Merging means the change is now company-approved.
