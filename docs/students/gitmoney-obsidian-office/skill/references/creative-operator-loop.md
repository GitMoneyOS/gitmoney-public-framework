# Creative Operator Loop

How creative work moves through the Office without dying in a drafts folder.

## The Loop

```text
Spark -> Capture -> Develop -> Blueprint -> Build -> Ship decision -> Evidence -> Next
```

- **Spark**: an idea hits. Do not develop it on the spot.
- **Capture**: one note in `00-inbox/`, six frontmatter fields, back to your day. Cost: 90 seconds.
- **Develop**: at review, promising sparks move to `20-creative-projects/` using the creative project template. Add the "This reminds me of..." link - creative leverage lives in collisions between notes.
- **Blueprint**: before any agent builds anything, write the project blueprint: what it is, who it serves, what done looks like, what is out of scope. This is the vibe-coding firewall.
- **Build**: scoped agent briefs per the workbench contract. One brief per deliverable, not one brief per dream.
- **Ship decision**: shipping is an owner decision recorded in `80-decisions/`, with a promotion target - content, repo, offer, or archive.
- **Evidence**: the shipped thing, linked. Screenshots, URLs, the artifact itself.
- **Next**: the retro line in the project note: what worked, what to reuse. Reusable fragments graduate to `50-sops-and-prompts/`.

## Rules Of The Loop

1. Capture is always allowed; development is scheduled. The inbox protects your focus from your own enthusiasm.
2. Three active creative projects maximum - matching the board's WIP limit. The rest wait in Backlog honestly.
3. A creative project with no movement in 30 days appears in the Stale Notes base and gets a decision: revive, archive, or hand off. No zombie projects.
4. Content derived from a project links back to it. The project is the trunk; posts, videos, and threads are branches in `40-content-and-media/`.
5. Done is defined in the blueprint before the build starts, never renegotiated mid-build to feel better.
