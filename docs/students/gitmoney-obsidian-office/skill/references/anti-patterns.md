# Anti-Patterns

## Purpose

Identify and explain common AI Office anti-patterns to avoid.

## Owner

Red Pillar and Hitsuyo Aku, under KnowTheLedge.

The ten ways students kill their vault, and the counter for each. Check this list at every weekly review.

## 1. Setup Paralysis

Tweaking folders, themes, and plugins for days before capturing one real idea.

**Counter:** the 30-minute win in `START_HERE.md` comes before any customization. One real note beats a perfect empty system.

## 2. Folder Overwhelm

Inventing a folder for every topic until filing requires a decision tree.

**Counter:** the lanes are fixed. Ten numbered folders, four utility folders. Links do the organizing that folders cannot.

## 3. Metadata Fatigue

Twenty frontmatter fields, most empty, all guilt.

**Counter:** six required fields. Anything beyond that must earn its place in a template or get cut.

## 4. Fake Linking

Linking every note to every note because a video said links are the point.

**Counter:** one real "This reminds me of..." link beats ten reflex links. A link must say something a folder cannot.

## 5. Input Hoarding

Clipping fifty articles a week and reading none. The vault becomes a landfill with good lighting.

**Counter:** the weekly review empties the inbox: file it, link it, or archive it. Archiving an unread clip is success, not failure. The Stale Notes base shows the hoard honestly.

## 6. Plugin Bloat

Installing every community plugin with stars.

**Counter:** v1 runs on core plugins only. A plugin is added when a named, recurring task demands it - never before.

## 7. AI Amnesia

Every agent session starts from zero; the agent re-learns the vault, repeats old mistakes, contradicts old decisions.

**Counter:** `_agent-context/AGENT_CONTEXT.md` is read at session start; `MEMORY_LOG.md` is appended at session end. Decisions live in `80-decisions/` where agents can find them.

## 8. Vibe-Coding Without Blueprint

Asking an agent to "build the app" from a feeling.

**Counter:** no AI coding or deployment before a `PROJECT_BLUEPRINT.md` or `SPEC.md` exists. The blueprint is the cheap place to be wrong.

## 9. Agent Jobs Without Scope

"Clean up my vault." "Make this better." "Handle my content."

**Counter:** the workbench contract. No brief, no job. A brief without scope and done-when is bounced back with questions.

## 10. Promoting Private Memory As Public Record

Pushing raw notes, doctrine, half-thoughts, or client details to GitHub because they look done.

**Counter:** the promotion rules. Everything is `private` until classified otherwise; promotion requires the asset test plus owner approval plus the public-safety check. Private memory leaking into the official record is the one mistake this whole system exists to prevent.
