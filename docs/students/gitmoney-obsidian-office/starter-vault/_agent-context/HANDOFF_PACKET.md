# HANDOFF PACKET

Use this when work moves between agents, between platforms, or from agent to owner mid-job. A handoff with no packet is a restart in disguise.

Copy per handoff:

```markdown
## Handoff: [job name] - YYYY-MM-DD

### From / To

[agent or platform] -> [agent, platform, or owner]

### The Job

- Brief: [[brief-link]]
- Board lane right now: 
- One-sentence state: where the work actually is, honestly.

### What Is Done

- Artifacts produced so far, with locations:
- Evidence attached so far:

### What Is Not Done

- Remaining done-when items from the brief:
- Known problems or surprises found:

### Decisions Made And Pending

- Decided during this work (link `80-decisions/` entries):
- Waiting in the owner decision queue:

### Exact Next Action

The single next step the receiver should take. Not a theme - an action.

### Context The Receiver Cannot Guess

Anything learned the hard way: dead ends tried, constraints discovered, owner preferences observed. This section is the reason the packet exists.
```

Rule of thumb: write the packet for a competent stranger. If your receiver needs the conversation history to proceed, the packet is not finished.
