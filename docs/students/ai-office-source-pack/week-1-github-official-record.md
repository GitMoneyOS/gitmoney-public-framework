# Week 1: GitHub Is The Official Record

Recorded: 2026-06-11
Audience: GitMoneyOS students
Layer classification: GitHub record

## Lesson Objective

Explain GitHub as the place where approved business work becomes a receipt.

GitHub is not just a developer tool. In GitMoney, it is the official record for work that has been approved, reviewed, and made reusable.

## Worked Example

Raw thought:

```text
I need to clean up how I answer the same customer question every week.
```

Official-record candidate:

```text
Issue: Create reusable customer-question SOP
Owner: student
Why it matters: the answer is repeated often enough to become a business asset
Done when: SOP draft exists, evidence is attached, owner decides whether to promote it
```

## Proof Artifact

Create one issue-ready task or official-record note with:

- owner,
- purpose,
- done-when proof,
- promotion target,
- next review point.

## Promotion Boundary

GitHub receives approved work. It does not receive raw private memory, sensitive notes, secrets, client data, or unfinished thoughts.

## Completion Checklist

- [ ] I can explain repository, issue, commit, and pull request in business language.
- [ ] I can name what belongs in GitHub and what stays private.
- [ ] I created one official-record candidate.
- [ ] I can explain why the candidate matters to ownership, evidence, or reuse.
