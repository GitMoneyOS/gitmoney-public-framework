# Week 3: Agents Are Workbenches

Recorded: 2026-06-11
Audience: GitMoneyOS students
Layer classification: GitHub record

## Lesson Objective

Use an agent through a scoped brief instead of vague prompting.

An agent platform is a workbench. It can help you draft, organize, test, and improve work. It should not become a hidden decision-maker.

## Worked Example

Bad prompt:

```text
Make this better.
```

Agent brief:

```text
Source: pricing objection note
Job: draft a one-page SOP for answering the objection
Boundaries: no public posting, no client names, no promises
Deliverable: SOP draft in markdown
Evidence: checklist showing owner, approval path, evidence, reuse path, and lifecycle
```

## Proof Artifact

Create one agent brief from a source note and produce one draft output with evidence required.

## Promotion Boundary

Agent output is a draft until the owner approves it. No brief means no job.

## Completion Checklist

- [ ] I wrote one bounded agent brief.
- [ ] I named the source note or issue.
- [ ] I named forbidden actions.
- [ ] I required evidence in the output.
- [ ] I can explain why the agent did not get authority to publish, merge, delete, or change permissions.
