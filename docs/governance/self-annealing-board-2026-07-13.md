## Card Controls

### SA-01 Restore the proof loop

- Value path: trustworthy activation data and visible self-healing.
- Deterministic work: workflow syntax, label matching, required fields, and link checks.
- Judgment work: what a public receipt may disclose.
- Stop condition: no workflow receives write permissions beyond its stated job.
- Rollback: revert the focused workflow and template changes.

### SA-02 Establish one founder front door

- Value path: score completion and qualified diagnostic demand.
- Sacred cow: founders retain control without needing developer identity.
- First asset: an owner-approved readiness snapshot or official asset receipt.
- Stop condition: no public score or sensitive evidence is required.
- Rollback: retain the current public proof package as the deeper verification route.

### SA-03 Complete the public trust baseline

- Value path: risk reduction and contribution readiness.
- Sacred cow: show the workflow while protecting private source material.
- Stop condition: do not invent a license, contact address, live security state, or approval.
- Rollback: keep owner-gated files as explicit decisions rather than false completion.

## Definition Of Done

- [ ] Output exists on a focused branch.
- [ ] Acceptance criteria pass with fresh deterministic evidence.
- [ ] No secrets, client data, private source paths, or unsupported claims are present.
- [ ] Human review is requested for public positioning and official acceptance.
- [ ] Live GitHub settings are reported separately from local files.
- [ ] Every blocked item names the exact owner decision.
- [ ] The healing dashboard records the cycle after acceptance.

## Pattern Memory Calibration

Closest pattern: Mothership First-Principles Scrumban Self-Heal.

- Failure drift prevented: adding more structure while broken routing and measurement remain.
- Success move: repair the smallest control surface, close with evidence, and leave an owner decision queue.
- Sacred cow protected: private source doctrine never becomes public by accident.
- First official asset: this reviewable self-annealing change packet.
- Evidence standard: diff, deterministic checks, live setting receipts, and owner review.
- Lifecycle: monthly governor review and a fresh self-annealing cycle after measured activation data.
