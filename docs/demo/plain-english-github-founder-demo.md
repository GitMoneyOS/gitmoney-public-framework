# Plain-English GitHub Founder Demo

## Purpose

Show non-technical founders how GitMoneyOS uses GitHub as an AI Office: a place to organize business knowledge, track work, approve changes, and keep evidence.

## The Five-Minute Story

GitMoneyOS is not GitHub for developers. It is GitHub as business infrastructure for founders using AI.

Most AI-enabled businesses create work faster than they create proof. GitMoneyOS helps answer:

- What changed?
- Who approved it?
- Where does this knowledge live?
- What is still risky?
- What can an AI agent do, and where does a human approve it?

## Plain-English Map

| GitHub Word | Founder Translation |
| --- | --- |
| Organization | the company AI Office building |
| Repository | a business vault for one kind of knowledge |
| Issue | a trackable task, risk, or decision |
| Branch | a draft room |
| Pull request | an approval packet |
| Merge | the change becomes official |
| Commit | a timestamped receipt |
| CODEOWNERS | the approval chain of command |
| Branch protection | a lock on important records |

## What To Show First

1. Open the public framework repo.
2. Read the first paragraph of the README.
3. Open the merged public positioning PR.
4. Explain that the PR is the approval packet.
5. Open a public issue and explain that it is a business task, not just a bug.

## Private AI Office Map

GitMoneyOS also uses private repos for internal work:

- `gitmoney-os`: source doctrine and operating system
- `gitmoney-audit-system`: intake, reports, and Exit-Ready Score
- `gitmoney-client-delivery`: setup sprint and handoff
- `gitmoney-content-engine`: public content and case files
- `gitmoney-saas`: dashboard and product layer

These private rooms are not public because founder IP, client delivery, and internal doctrine should not be exposed by default.

## Founder Takeaway

The point is not to make the founder technical.

The point is to give the founder a control room where AI-assisted work becomes owned, reviewable, and easier to hand off.

## Try It

Use the public issue template to describe one business task in plain English:

- What needs to change?
- Why does it matter?
- Who owns the decision?
- What proof would show it is complete?

That is the first step from scattered work to governed evidence.
