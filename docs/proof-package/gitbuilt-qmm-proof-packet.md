## The QMM Proof Cards

### KAREN — Manuscript-Structure Scanner

KAREN is a public, MIT-licensed, open-core scanner for DOCX, PDF, and Markdown manuscripts. Its public code diagnoses four kinds of structural defects; the repair pipeline is intentionally not included in the public release.

**Inspect it yourself:** open the [README and source](https://github.com/respectfulnrespected59-source/karen), review the [MIT license](https://github.com/respectfulnrespected59-source/karen/blob/master/LICENSE), and run the documented CLI if you want to test the scanner.

**Why it belongs here:** it shows a real product boundary. A third party can see the public capability, the license, the source files, and the line between the free diagnostic tool and the withheld repair workflow.

**Claim boundary:** this card does not claim that the paid repair pipeline is public, that every manuscript will be repaired, or that a scan establishes editorial quality.

### DCC — Digital Creative Contractors, Phase 0

DCC is a public QMM × GitMoney project surface for a proposed freelance-platform model. Its repository presents the Phase 0 landing page, pitch deck, launch tracker, and open Phase 1 workstream issues in one inspectable place.

**Inspect it yourself:** compare the [repository](https://github.com/respectfulnrespected59-source/dcc) with the live [landing page](https://respectfulnrespected59-source.github.io/dcc/), [pitch deck](https://respectfulnrespected59-source.github.io/dcc/deck.html), [launch tracker](https://respectfulnrespected59-source.github.io/dcc/tracker.html), and [open issues](https://github.com/respectfulnrespected59-source/dcc/issues).

**Why it belongs here:** it makes an early-stage build legible as more than an idea. The public record shows the proposition, the current Phase 0 surfaces, the proposed next build, and work that can be claimed.

**Claim boundary:** this card does not claim that DCC is a launched marketplace, that its proposed payment or escrow system is live, or that the repository is an open-source project. Its public README identifies the surface as Phase 0 and its Phase 1 work as proposed.

### QMM Links — Public Point Of Entry

QMM Links is a deliberately simple public links surface: an `index.html` page in a public repository, served through GitHub Pages.

**Inspect it yourself:** open the [repository](https://github.com/respectfulnrespected59-source/qmm-links) to review the page source, then open the [live surface](https://respectfulnrespected59-source.github.io/qmm-links/).

**Why it belongs here:** a proof record also needs a clear front door. This card shows a simple, discoverable public entry point without asking the reader to infer a larger product claim.

**Claim boundary:** this card establishes a public links surface only. It does not assert traffic, conversion, availability of every linked destination, or an open-source license.

## What A Third Party Can Verify

On 2026-07-20, read-only checks confirmed that the three repositories above were publicly reachable. The DCC landing page, deck, and tracker, plus the QMM Links page, returned successful HTTP responses at that time.

The strongest proof is not a summary on this page. It is the route from this card to the underlying public work:

1. Open the repository or live surface.
2. Compare the stated scope with the visible code, documentation, page, or issue list.
3. Keep the conclusion inside the claim boundary printed on that card.

## What This Collection Shows

Together, these examples make three kinds of work visible:

- a bounded tool with an explicit license and public/private capability line;
- an early project with public planning, presentation, and workstream evidence; and
