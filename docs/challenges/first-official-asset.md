# Seven-Day First Official Asset Challenge

Status: launch-ready draft pending dual-owner review
Classification: Public
Owner: Red Pillar and Hitsuyo Aku, under KnowTheLedge
Audience: nontechnical founders and operators using AI-assisted work
Purpose: create one governed asset without exposing private source material
Update cadence: review after every cohort or 30 days of public receipts
Approval path: both owners approve launch and claim language through a pull request

## The Promise

In seven days, turn one scattered piece of useful work into an asset with an owner, approval path, evidence, reuse path, and lifecycle.

This challenge does not require you to publish the asset. Private work can remain private.

## Day 1: Choose The Work

Pick one item that already matters:

- decision
- operating procedure
- scorecard
- approved prompt or agent rule
- product or delivery specification
- reusable public contribution

Done when: you can explain the business job in one sentence.

## Day 2: Name The Owner And Boundary

Name one accountable person or role. Classify the source and final asset as public, internal, confidential, restricted, secret, or client-owned.

Done when: you know what must never enter a public issue or AI chat.

## Day 3: Create A Bounded Work Order

State the goal, approved source, allowed actions, forbidden actions, expected output, required evidence, and stop condition. Use a human or AI workbench only within that boundary.

Done when: another person can tell what the workbench may and may not do.

## Day 4: Draft The Asset

Create the smallest version that can perform the real job. Do not add features that are not needed for first use.

Done when: the output exists in a reviewable form.

## Day 5: Review And Test

Use a human reviewer for judgment. Use deterministic checks for links, calculations, format, required fields, or repeatable tests.

Done when: the reviewer can approve, edit, reject, or respond using visible evidence.

## Day 6: Promote The Approved Record

Move only the approved asset and minimum necessary evidence into the correct official repository. Keep raw notes, private context, credentials, and client material out.

Done when: the current version and approval receipt are inspectable by the people who are allowed to see them.

## Day 7: Close The Lifecycle

Name the reuse path and next review trigger. If the asset and evidence are safe for public use, optionally file a [First Official Asset Receipt](https://github.com/GitMoneyOS/gitmoney-public-framework/issues/new?template=first-official-asset.yml).

Done when: the asset has a real next use and a review, archive, or retirement point.

## Success Measure

Primary measure:

> Verified first official assets divided by qualified challenge starts.

Quality measure:

> Assets reused or updated within 30 days divided by verified first official assets.

Stars, likes, prompts, and file counts are not activation evidence.

## Safety Rules

- The private path is the default.
- Public receipts require explicit consent and a public-safe evidence URL.
- Never publish exact score evidence, client identity, credentials, access detail, private links, or confidential strategy.
- A contribution story requires permission from every person identified.
- No income, security, investment, valuation, or transaction outcome is implied.

## First Checkpoint

Finish this sentence:

> The asset is ______. Its owner is ______. Its approved evidence will live ______. It will be reused for ______ and reviewed again ______.
